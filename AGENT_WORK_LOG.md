### Причина, почему на живом сайте не было фото (разобрано отдельно)
Старая версия использовала `<image-slot>` — компонент из инструмента для прототипов,
который подтягивает фото из служебного файла `.image-slots.state.json` через
`window.omelette`-мост. Этот мост существует только внутри самого инструмента
прототипирования; на обычном GitHub Pages его нет, а сам файл `.image-slots.state.json`
не был закоммичен ни разу. Поэтому слоты для hero-фото, about-фото и 4 фото врачей
были пустыми с самого начала существования сайта — это никак не связано с сегодняшними
изменениями, а вскрылось при разборе.
